Projeto

Informações Sobre o Projeto

O objetivo do projeto é praticar os conceitos vistos em sala para produzir um (protótipo) sistema de visualização interativa de dados para análise de uma conjunto de dados real. O sistema produzido deve ser um sistema Web implementado em D3. Para realizar este projeto os alunos deverão se reunir em grupos de até 3 integrantes (alunos da pós-graduação deverão fazer projetos individualmente). Note que o objetivo final é buscar um melhor entendimento dos dados. A visualização deve ser um instrumento para alcançar esse objetivo. Isso significa, que os projetos serão avaliados não somente na visualização produzida (considerações de design), mas também em fatos interessantes que foram obtidos a partir das visualizações.

Informações Sobre a Avaliação

A avaliação do projeto será baseada em três critérios: qualidade do projeto e resultados alcançados, apresentações do projeto e pontualidade. Durante a realização do projeto, os grupos deverão apresentar fazer 3 apresentações do seu projeto: Uma apresentação inicial defendendo sua proposta (dataset a ser analisado e ideias inicias sobre a análise que será feita), uma apresentação intermediária para informar a classe sobre o progresso na execução do projeto e uma apresentação final que deverá constar de uma demonstração do sistema implementado e resultados obtidos. Cada uma destas apresentações será parte da avalição. A fim de auxiliar a realização dos projetos, cada grupo será acompanhado individualmente em aulas de acompanhamento. As aulas de acompanhamento também serão parte da avaliação. Os integrantes do grupo deverão estar presentes nas aulas de acompanhamento, que constaram de discussões sobre o estado atual e definições de metas para serem alcançadas. O dia de cada apresentação será um dia de entrega. Nesses dias os slides de apresentação e código da aplicação devem estar salvos no repositório git do projeto.

Instruções de Entrega e Execução

Cada grupo deverá criar um repositório público no GitHub compartilhar esse repositório com o usuário visualizacao-ufpe. O nome do repositório deverá seguir o formato NOMEDOPROJETO-visualizacao-2017-1.

Todos os materiais produzidos (textos, apresentações, código e etc) devem estar disponíveis no repositório. Os dados devem estar disponíveis para download, a fim de que os resultados obtidos possam ser replicados. O repositório deve conter uma página HTML descrevendo o sistema desenvolvido e instruções de uso (como rodar e utilizar o sistema). Tal página deve ser feita com o GithubPages. Além disso, a página deve conter emails para os slides de apresentação e demo do projeto.
